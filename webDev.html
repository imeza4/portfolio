Improving Data Visualization for a Company

client/class project | ux design | data visualization | 2024


The Project


As a class project, I analyzed different companies' products and then practiced the workflow of developing an interactive interface. I chose a concept and built it with React JS in order to create an interface with filtering components.

Project Scope


I chose to do a bookstore-esque concept for the web application. The book application informs users about the books available. Users will be able to interact with the site which is handled by React state variables which leads to easier navigation.

Features of Application


In my application, there are different features that are available to users created using React components, props, and states.

Users will be able to:

  • filter books based on page count, genre, and price.
  • sort books alphabetically and by price high to low/low to high
  • add books to their cart.
  • see the total price of items added to the cart.
  • clear items from the cart.

Architecture


I have the following component files:

  1. BookButton.js: All of the logic involved in developing the button that users press to add to their cart.
  2. BookItem.js: In the Book Item component, the props and the states ensure the filtering and sorting functionalities are connected to the products that are displayed on the home page.
  3. Cart.js: All the logic incorporated to build the cart happens in this file.
  4. ClearButton.js: Similar to the book button, everything in this file is used to clear the cart.
  5. FilterButton.js: In this file, is where all the logic for the filtering section is placed.
  6. SortButton.js: The buttons used to sort have their logic within this file.
